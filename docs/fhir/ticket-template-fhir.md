# Plantilla de ticket técnico FHIR

## Título

`FHIR-xxx — <resumen corto>`

## Objetivo

Describir qué problema resuelve el ticket y por qué se hace ahora.

## Contexto

- hallazgo que origina el ticket;
- fase a la que pertenece;
- dependencia con otros tickets o ADRs.

## Alcance

- qué entra;
- qué capas se tocan;
- qué archivos o zonas del repo probablemente cambian.

## No alcance

- qué queda explícitamente fuera;
- qué no debe mezclarse en el mismo PR.

## Riesgo

- bajo / medio / alto;
- riesgo técnico;
- riesgo funcional.

## Criterio de aceptación

- resultado verificable por código + tests + docs.

## Validación mínima

- lint/test/build si aplica;
- pruebas unitarias;
- pruebas de integración;
- validación manual de UI si aplica.

- `README.md`
- `docs/fuente-de-verdad-operativa.md`
